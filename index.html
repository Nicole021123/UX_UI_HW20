Website Redesign

UX Research, UI Design

As a result of collective deliberation, ProjectStay emerged as the unanimous choice for a group project. We arrived at this decision based on the non-profit organization's lack of responsive web design for mobile devices. Furthermore, the current website suffered from a user-unfriendly and unreliable content layout, posing challenges for visitors in comprehending the services provided by ProjectStay.

View Case Study

Problem

Project Stay was designed to assist individuals with securing a post secondary education. We have observed the website has misleading information and lacks clear navigation which is causing individuals to not utilize the Project Stay resources. How might we improve the design navigation and content layout so that users can easily determine the onboarding process and follow through with services.

Solution

We believe improving Project Stay’s website will make a student’s process easier to return to College and allow them to achieve their academic goals.

My Design Process

Persona

We wanted to form a deeper understanding of our users' goals, needs, experiences, and behaviors. So, we created a persona that aligned with the non-profits target audience and continued to update throughout our project. We iterated on our persona after further research of the non-profit and conducting usability testing.

Appearance, Content and Navigation

We have observed the website created confusion through a lack of clear navigation which is causing individuals to not utilize the non-profit resources.

Card Sorting

To make sure that the sites information architecture is aligned with user expectations, we had 3 remote card sorting sessions using Figjam. Our goal was to find smaller, intuitive groupings for the existing 8 categories. After organizing the data, we ended up with 4 clearly defined main categories with 2 subcategories for a secondary navigation.

Usability Testing

I created a fully-functional, high-fidelity prototype of the new flows using Figma. At the same time, we started recruiting subjects for the test who fit our criteria. We did 2 usability tests in the first round and 2 after iterating on the issues that we’ve identified:

Issue 01
Content
Through initial usability testing we were able to quantify the pain points we found through our Heuristic Evaluation. We followed our analysis with a redline and annotation of the site to ensure we outlined the correct information architecture needed for transparency.
Solution 01

Restructure of Navigation
Based on our research conducted we identified three pain points a user had and determined the biggest impact would be to restructure the content layout.

    1. Make information clearer so it requires less clicks to find it.
    2. Allow for the site to guide the user and not make them find information on their own.
    3. Add hours of operation.
